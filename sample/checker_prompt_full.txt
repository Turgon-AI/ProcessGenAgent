- [ ] Row 1 Direction: First row flows LEFT to RIGHT

- [ ] Row 3 Direction: Third row flows LEFT to RIGHT (if applicable)
- [ ] Alternating Pattern: Each subsequent row alternates direction from the previous row
- [ ] Arrow Direction: Arrows correctly follow the snake path, including transitions between rows

### 4. Content Accuracy (Critical - Must Match Source Document Exactly)

For EACH step, verify:

#### Step Description Match
- [ ] The full description text matches the source content document WORD FOR WORD
- [ ] No paraphrasing, summarizing, or rewording has occurred
- [ ] No words have been added or removed

#### Step Name Match
- [ ] The step name accurately reflects/summarizes the description
- [ ] Step names are consistent with the action described

#### System & Performer Match
- [ ] The system listed matches the source document exactly (e.g., "Oracle Expenses", "Oracle Payables")
- [ ] The performer listed matches the source document exactly (e.g., "Employee", "Employee Manager", "Expense Auditor")
- [ ] Format is exactly: System | Performed By (in italics)

### 5. Visual Elements

#### Step Node Format (Critical)
- [ ] NO CARDS: Steps must NOT be displayed as cards or card-style containers

- [ ] No filled background shapes behind individual steps
- [ ] No divider lines within or between steps
- [ ] Steps should use the standard open node format: icon positioned above text layers, without any enclosing container or visual separators

#### Icons
